# Output Checklist

Use this checklist to review final answers after meaningful Figma work.

## Runtime Self-Check

- Trigger: says why Figma work was appropriate.
- Route: names the skill path used.
- Stage: includes page/component stage when relevant.
- Evidence: cites metadata, inventory, component audit, resize audit, layout audit, or screenshot.
- Closure: says whether QA is complete, pending, or not applicable.

## Governance Output

- Scope is clear.
- Findings include evidence.
- Priorities are meaningful.
- Recommendations are actionable.
- Confirmation points are separated from safe actions.
- The answer does not claim visual QA unless screenshot/layout audit happened.

## Component Output

- Page stage affects strictness.
- Integrity and resize concerns are not mixed together.
- Source component and representative instance are both considered when relevant.
- Broad replacement waits for user confirmation.

## QA Output

- Blocker/major findings are not ignored.
- Screenshot review is mentioned for meaningful visual updates.
- Remaining risks are explicit.
